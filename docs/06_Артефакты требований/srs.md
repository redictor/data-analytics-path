# Спецификация требований

Спецификация требований к ПО (software requirements specification, SRS) - структурированный набор требований (функции, производительность, ограничения разработки, атрибуты) ПО и его интерфейсов (связей с внешними системами). Необходимы Разработчикам, Тестировщикам для верификации, Администраторам Системы для её поддержки, Аналитикам для анализа.

Спецификация требований к ПО обычно содержит:

- требования к внешним интерфейсам (external interfaces)  
  Определяют все входные и выходные данные  

- требования к функциям продукта (functions)  
  Определяют все возможности, которые есть в программном продукте  

- требования к дизайну/юзабилити (usability)  
  Определяют требования к юзабилити  

- требования к производительности (performance)  
  Определяют числовые требования, предъявляемые к программному обеспечению или к взаимодействию человека с программным обеспечением в целом  

- требования к логической структуре БД (logical database)  
  Задают логические требования для любой информации, которая должна быть помещена в базу данных  

- ограничения проектирования/разработки (design constraints)  
  Указывают ограничения на проектирование системы  

- соответствие стандартам (standard compliance)  
  Указывают требования, вытекающие из существующих стандартов  

- атрибуты программного продукта, как то:  
  - Надёжность  
  - Доступность  
  - Безопасность  
  - Ремонтопригодность  
  - Переносимость (кроссплатформенность)
